POST /auth/register, POST /auth/login → trả JWT. Các request sau gắn token; guard kiểm tra vai trò.

B. Flow — Đặt sân thường

POST /bookings (hoặc /bookings/hold). Kiểm tra slot trống → tạo booking pending + khoá slot → sinh hoá đơn → chuyển sang thanh toán. NV/Admin xác nhận thanh toán → confirmed.

D. Flow — Đặt hàng online

POST /orders (public, token tuỳ chọn để gắn user). Chọn giao hàng hoặc nhận tại chi nhánh; hệ thống chọn kho phục vụ, tính phí ship, sinh hoá đơn, rồi qua thanh toán.

E. Flow — Cổng thanh toán (VNPay / MoMo / SePay)

POST /payment/create sinh URL/QR. Người dùng được redirect về return; cổng gọi ipn (server-to-server) để xác nhận chắc chắn → cập nhật Payment + Invoice.

sequenceDiagram autonumber actor U as Khách participant BE as BadmintonHub API participant GW as Cổng thanh toán U->>BE: POST /payment/create (invoiceId, method) BE->>BE: Tạo Payment = pending + ký chữ ký BE-->>U: Trả paymentUrl / QR U->>GW: Quét QR / nhập thẻ GW-->>U: Redirect /payment/.../return GW->>BE: IPN (server-to-server) + chữ ký BE->>BE: Verify chữ ký + số tiền alt Hợp lệ & thành công BE->>BE: Payment = success, Invoice = paid BE-->>GW: 200 OK (đã ghi nhận) else Sai chữ ký / thất bại BE->>BE: Payment = failed BE-->>GW: Trả mã lỗi end

H. Flow — Nhập kho qua đơn mua (Purchase Order)
